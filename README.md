# 트러블 슈팅

## 메인 씬 증발 사건

### 🔍 문제 상황
Develop branch에 코드를 병합하는 과정에서 **메인 씬(Main Scene)이 사라지는 문제**가 발생함.
프로젝트 구조상 서로 연동되는 정보가 많았고, 각자 작업한 브랜치를 Develop에 올리고 받아오는 과정에서 충돌이 누적됨.
이 과정에서 메인 씬 파일이 정상적으로 병합(Merge)되지 못하고 삭제되는 현상이 발생함.

---

### 🧪 원인 분석

  - Unity 씬 파일은 텍스트 기반이라 병합(Merge) 시 충돌이 자주 발생함.
  - 평소에 오류 해결 과정에서 2명 이상의 팀원 같은 씬을 수정한 것이 원인.
  - 충돌 해결 과정에서 실수로 씬 파일이 누락됨.

---

### 🔧 해결 방법
  - GitHub 저장소에서 문제가 발생하기 직전 commit으로 **롤백**하여 메인 씬을 복구.
  - 이후 같은 문제가 재발하지 않도록 규칙 마련.
    - 오류 해결 과정에서, 같은 씬이나 동일한 C# 스크립트를 수정해야 할 때는 **화면 공유를 통해 한 명의 화면으로 작업**
    - 동시에 수정하지 않고 **한 명이 수정 완료 후 다른 사람이 이어서 코드 추가**하는 **순차 작업 방식**으로 진행.

---

### ✅ 결과 및 이점
  - 메인 씬을 정상적으로 복구하여 프로젝트 진행을 이어갈 수 있었음.  
  - 충돌 상황에 대한 대응 방식을 정립하여, **협업 과정에서 충돌로 인한 작업 손실을 예방**.  
  - Git 롤백 경험을 통해, 문제 발생 시 **버전 관리 도구를 활용한 복구 능력**이 강화됨.

---

### 📝 회고
브랜치 전략이 명확하더라도, 오류 해결 시 협업 규칙이 흐트러지면 충돌은 쉽게 발생할 수 있음을 체감함.  
화면 공유와 순차 작업을 도입하면서, 충돌 발생 가능성을 크게 줄였고 협업 효율도 높아짐.  
앞으로는 공용 파일 수정을 최소화하고, 되도록이면 **작업 단위 분리 → 병합** 절차를 지키는 것이 중요하다고 느낌.
